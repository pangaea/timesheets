jQuery SexySelect v. 0.3

Author : Nico VanHaaster Updated: 23/11/2010

The jQuery SexySelect is a simple plugin written for educational purposes to allow for easier (and frankly prettier) manipulation of Single or Multiple select boxes.

This plugin will only work with select boxes and no other element types.

Features

  • Skinned using jQuery UI themes 1.8.6+
  • Collapsable select region
  • Allow user to add & remove options
  • Single, Multiple & No Selection modes
  • Auto Sorting
  • Drag & Drop Sorting (with Auto Sorting disabled)
  • Events On Item Creation & Items Syncronization
  • Custom Header Text
  • Simple item count & validation methods
  • Watermark Entry Text
  • Custom CSS Styles such as height, width, and background display.

Options

The sexy select can be initialized with the following options. (note: case sensitive)

Option Name Possible Values Description
width Any width equivilant value
default: 230 (note any value <150 will be set to 150)		 Sets the width of the entire select container.
height Any height equivilant value
default: 200		 Sets the height of the entire select container
background CSS equivilant to the containers background
default: #fff		 Sets the background CSS of the entire container
title Any text to be displayed as the title bar text.
default: Multi-Select		 Sets the title bar text
showTitle boolean : true\false
default: true		 Allows you to show or hide the Title bar, this will also hide the collapse feature
allowInput boolean : true\false
default: true		 Allows you to show\hide the add new option input bar.
addValueMaxLength int
default:50		 Sets the maximum length of the add new option input control. This can be set to any int value
defaultInputText text
default: 'add new option'		 Sets the watermark text of the add new option value. This text is displayed when the new option input control has lost focus with a blank value
autoSort boolean: true\false
default: false		 Automatically sorts the list to be in alphabetical order. If the autosort is set to true the sortable drag & drop features are disabled.
minSize int
default: 0		 Sets the minimum size option for the validation method. This is used when the validation method is called, see the validation method
autoExpand boolean : true\false
default: false		 Currently not used, will be used in future releases to make the select box increase and decrease as items are added\removed
allowDelete boolean: true\false
default: true		 Allows the delete icon to appear and the user to remove the selected item.
allowCollapse boolean: true\false
default: true		 Allows the container (minus the title) to expand \ contract with clicking the title bar. If the option 'showTitle' is set to false this option will be skipped
selectionMode values:
  • none
  • single
  • multiple
default: none 		The three states drastically change the functionality of the list.
 - none : Removes the ability to select any items. Nicely used for sorting, adding & deleting items
 - single : Shows radio buttons and allows for single selection
 - multiple: Shows checkboxes and allows for multiple item select
Note that if the <select> element has the attribute 'multiple' and the selectionMode is set to none then the selectionMode will be overwritten to multiple
expanded boolean: true\false
default: true		 Allows you to initilize the widget in expanded or collapsed state. Setting expanded to false will start the control collapsed
allowDebug boolean:true\false
default:false		 Used only to debug, will display both the widget and the original select element
nooptionstext string
default: 'no options added...'		 Sets the no options text when there are no options added to the list

Methods

Method Name Sample Description
searchItem alert($('#element').sexyselect('searchItem','Value1'))
try sample		 Allows searching of all the options already added and returns true or false with the item found.
destroy $('element').sexyselect('destroy'); Destroys the widget and removes it from the DOM
validateSize alert($('element').sexyselect('validateSize')
try sample		 Returns true or false if the minSize of the list is met. See minSize option for its description.
totalItems alert($('element').sexyselect('totalItems'))
try sample		 Returns the total count of all the options in the select list (including added items)
showError $('#element').sexyselect('showError','This is text',2500) Shows the error message bar with your custom set Text and Timeout.

Events

Event Name Sample Description
onitemcreated $('#element').sexyselect({ onitemcreated: function(newOption,initElem){ .... }); This event is fired after each element is created and returns both the new option created (<option />) and the initial select object on the page.
onItemsSync $('#element').sexyselect({onItemsSync: function (initElem) { ... } ); This event is fired after all the items are syncronized. Syncronization happens through most events, created, deleted, sorted and moved returns the initial element
onItemSelected $('#element').sexyseelct({onItemSelected:function(element,options){
  alert(element.val());
});		 This event is fired after the item is selected, the event passes in the element selected (input type=checkbox|radio) and also an array of the options in the array.
To parse the options use the syntax similar to $.each(options,function(idx,i){ alert(i.val()); });
onitemcreating $('#element').sexyselect({onitemcreating: function(object, EventArgs) { return EventArgs }); This function is fired before an item is created. This gives you a first chance to validate the text that will be added to the list. This event passes in both the input text object (validate with object.val()) and also Event Arguments. These Event arguments are expected to be returned an contain 3 parameters as below.
EventArgs
EventArgs.create : boolean, default true. Changing this to false on return will cancel the creation of the item.
EventArgs.errorText: string, default ''. Set the error text to be displayed if the EventArgs.create is set to false.
EventArgs.timeOut: int, default: 2500. The timeout for the error window to hide itself.

Passing back no event Args will add the item regardless of your validation efforts, additionally blank errorText will also skip showing the error message.

Change Log

Changes in v0.3
  • Renamed option : _debugShowOriginal to allowDebug. This option also will show alert messages on errors through custom events.
  • Added option : nooptionstext which allows for you to change the text when no options are added to the list, default is 'no options added'
  • Added event : onItemSelect : runs a custom event when an option is selected, returns the element created and a collection of all the options in the list. Runs after the click event is handled
  • Added event : onitemcreating : runs a custom event before the item is created. This allows for custom validation. This event passes in the input text object and also the EventArgs, this event expects the EventArgs to be passed back. See the Events Tab for more info
  • Added method : showError : Shows the error bar with custom text that you pass in. Expects the text to display and the timeout to slide up. See the methods tab
